Specificity Hierarchy

Level Selector Type Specificity Example
0 !important Overrides all color: red !important;
1 Inline styles 1,0,0,0 style="color: red;"
2 IDs 0,1,0,0 #header
3 Classes, attributes, pseudo-classes 0,0,1,0 .nav, [type="text"], :hover
4 Elements, pseudo-elements 0,0,0,1 div, ::before
5 Universal selector 0,0,0,0 *

Calculating Specificity

/**
 * SPECIFICITY CALCULATION
 * 
 * Format: (inline, IDs, classes, elements)
 * Compare from left to right
 */

/* Examples */
*                    ‚Üí 0,0,0,0
div                  ‚Üí 0,0,0,1
div p                ‚Üí 0,0,0,2
.class               ‚Üí 0,0,1,0
div.class            ‚Üí 0,0,1,1
#id                  ‚Üí 0,1,0,0

/* More IDs always win over more classes */
#a                   ‚Üí 0,1,0,0  ‚úì WINS
.a.b.c.d.e.f.g.h.i.j ‚Üí 0,0,10,0 ‚úó loses

Specificity Examples

/* Which color wins? */

/* Specificity: 0,0,0,1 */
p { color: black; }

/* Specificity: 0,0,1,0 - WINS over element */
.text { color: blue; }

/* Specificity: 0,0,1,1 */
p.text { color: green; }

/* Specificity: 0,1,0,0 - WINS (has ID) */
#intro { color: red; }

/* Specificity: 0,1,1,0 */
#intro.text { color: purple; }

/* Specificity: 0,1,1,1 */
p#intro.text { color: orange; }

/* !important overrides all specificity */
p { color: pink !important; }  /* WINS */

Common Specificity Scenarios

/* SCENARIO 1: Same specificity - last one wins */
.button { color: red; }
.button { color: blue; }  /* WINS - comes later */

/* SCENARIO 2: Nested selectors */
.nav .item { color: red; }      /* 0,0,2,0 */
.nav-item { color: blue; }      /* 0,0,1,0 */
/* Red wins - higher specificity */

/* SCENARIO 3: Pseudo-classes count as classes */
a:hover { color: red; }         /* 0,0,1,1 */
.link { color: blue; }          /* 0,0,1,0 */
/* Red wins when hovering */

/* SCENARIO 4: Attribute selectors count as classes */
input[type="text"] { }          /* 0,0,1,1 */
.input { }                      /* 0,0,1,0 */
/* Attribute selector wins */

/* SCENARIO 5: :not() doesn't add specificity itself */
:not(.foo) { }                  /* 0,0,1,0 (just .foo) */
:not(#bar) { }                  /* 0,1,0,0 (just #bar) */

Best Practices

/**
 * SPECIFICITY BEST PRACTICES
 */

/* ‚ùå BAD: Over-specific selectors */
#header #nav ul li a.link { }

/* ‚úì GOOD: Low specificity, easy to override */
.nav-link { }

/* ‚ùå BAD: Using !important */
.button { color: red !important; }

/* ‚úì GOOD: Use more specific selector if needed */
.form .button { color: red; }

/* ‚ùå BAD: IDs for styling */
#submit-button { }

/* ‚úì GOOD: Classes for styling */
.submit-button { }

/* ‚ùå BAD: Inline styles */
<div style="color: red;">

/* ‚úì GOOD: External/internal CSS */
.highlight { color: red; }

/**
 * METHODOLOGIES TO MANAGE SPECIFICITY
 * 
 * BEM (Block Element Modifier):
 * .block__element--modifier
 * 
 * All selectors have same specificity (0,0,1,0)
 */
.card { }
.card__title { }
.card__title--large { }
.card--featured { }

‚ö†Ô∏è When to Use !important

  • Utility classes that MUST override: .hidden { display: none !important; }
  • Overriding third-party CSS you can't modify
  • Temporary debugging (remove before committing!)

Never use !important for: Regular styling, fixing specificity wars

Specificity Quiz

/* Which selector wins? */

/* Question 1 */
.list .item { color: red; }     /* 0,0,2,0 */
ul li { color: blue; }          /* 0,0,0,2 */
/* Answer: .list .item (red) - classes beat elements */

/* Question 2 */
#nav a { color: red; }          /* 0,1,0,1 */
.nav .link { color: blue; }     /* 0,0,2,0 */
/* Answer: #nav a (red) - ID beats any number of classes */

/* Question 3 */
body #content .data { }         /* 0,1,1,1 */
html body #content .data { }    /* 0,1,1,2 */
/* Answer: Second one - same ID/class count, more elements */

/* Question 4 */
.a.b.c { }                      /* 0,0,3,0 */
.x { }                          /* 0,0,1,0 */
/* Answer: .a.b.c - more classes */

üé§ Interview Questions

  • How is specificity calculated? Count IDs, classes, elements - compare left to right
  • What beats an ID? Only inline styles or !important
  • Does :not() add specificity? No, but its argument does
  • How to avoid specificity wars? Use BEM, keep selectors flat, avoid IDs
  • When is !important acceptable? Utility classes, overriding third-party CSS
